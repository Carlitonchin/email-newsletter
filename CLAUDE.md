# CLAUDE.md

Guidance for AI agents making **code** changes to this repo. For the daily
**content** pipeline (reading email → generating summaries), follow
[`INSTRUCTION.md`](./INSTRUCTION.md) instead.

## What this is

"Brief" — a **static** website that turns the user's tech newsletters into a daily
digest. Each kept newsletter becomes a blog-style **article** (summary + quiz); a
**day** groups up to 5 articles. Built with Vite + React 19 + TypeScript +
Tailwind v4 + shadcn/ui (radix-nova) + next-themes. Package manager: **pnpm**.

## The golden rule: data vs. code

- **Data lives in `content/`** as JSON, generated by the agent (see INSTRUCTION.md).
  The site renders entirely from it.
- **Never hardcode digest data in components**, and **never edit `src/`,
  `components/`, or `lib/` to add a day/article** — add JSON under `content/`.
- To change how the site *looks or behaves*, edit the code. To change *what it
  shows*, edit `content/`.

## Commands

```bash
pnpm dev                 # local dev server (HMR)
pnpm build               # tsc -b && vite build → dist/   (this is the type-check gate)
pnpm preview             # serve the built dist/
pnpm lint                # oxlint
pnpm newsletter:validate # validate everything under content/ (run before build)
pnpm newsletter:since    # compute the email window for the next pipeline run
pnpm newsletter:asset    # download an original's image into public/editions/<date>/<slug>/
pnpm gmail:list / :get / :auth   # read-only Gmail scripts (see scripts/gmail/README.md)
```

Always finish a change by running **`pnpm newsletter:validate && pnpm build && pnpm lint`**.

## Architecture

- **Data flow:** `content/editions/<date>/*.json` is bundled at build via
  `import.meta.glob` in `lib/content.ts` — no manifest, no runtime fetch. Add a file,
  rebuild. The loader validates each file and skips invalid ones loudly.
- **Schema source of truth:** [`lib/edition.ts`](./lib/edition.ts) — types
  (`Article`, `DayMeta`, `Day`) + validators (`validateArticle`, `validateDayMeta`).
  It is **pure TS (no Vite/React/Node)** so both the app and the `scripts/newsletter`
  CLI import it. Keep it that way.
- **Storage:** one folder per day. `content/editions/<date>/index.json` = day
  metadata + ordered article slugs; `content/editions/<date>/<slug>.json` = one
  article (file name **must** equal `slug`, and `date` must equal the folder).
- **Routing:** tiny hash router in [`lib/router.tsx`](./lib/router.tsx) (`useHashPath`,
  `navigate`, `Link`). Routes are matched in `src/App.tsx`. Three levels: `#/` (home,
  days) → `#/edition/<date>` (day) → `#/edition/<date>/<slug>` (article). Hash routing
  keeps it deployable to any static host with no rewrites.
- **Theme:** `next-themes` (`attribute="class"`); a pre-paint script in `index.html`
  avoids the flash. Toggle in `components/site/theme-toggle.tsx`.

## Where things live

| Path | What |
| --- | --- |
| `src/main.tsx` / `src/App.tsx` | Entry + route resolution. |
| `src/pages/` | `home`, `edition` (day), `article`, `not-found`. |
| `components/ui/` | shadcn primitives — **managed by the shadcn CLI** (see below). |
| `components/site/` | Header, footer, theme toggle, brand mark, category badge, ambient bg. |
| `components/home/` | Hero, day card. |
| `components/edition/` | Article card, blog body (`rich-text`), inline `quiz`. |
| `lib/` | `edition` (schema), `content` (loader), `router`, `format` (dates), `site` (config), `utils` (`cn`). |
| `scripts/gmail/` | Read-only Gmail fetch (auth, list, get). |
| `scripts/newsletter/` | `since` (window) and `validate` (schema gate). |
| `content/` | All site data (agent-generated). |

> App components live in the **root** `components/` and `lib/`, not `src/`, because the
> `@` alias points at the project root (`@/components`, `@/lib`). Pages/app-shell are in `src/`.

## Conventions & gotchas

- **Path alias `@` → project root.** It must stay defined in **both**
  `vite.config.ts` (`resolve.alias`) and `tsconfig.app.json` (`paths`). The root
  `tsconfig.json` paths do **not** apply to the app project that `tsc -b` compiles.
- **Strict TS build.** `tsc -b` runs with `noUnusedLocals`/`noUnusedParameters`
  (remove unused imports), `verbatimModuleSyntax` (use `import type` for type-only
  imports), and `erasableSyntaxOnly` (no `enum`/`namespace` — use `as const` + union
  types, as in `CATEGORIES`).
- **shadcn components:** add/update them with the shadcn skill / CLI
  (`pnpm dlx shadcn@latest add <name>`), don't hand-write primitives. Skill docs live
  in `.agents/skills/shadcn/`. Project config in `components.json` (style `radix-nova`,
  base `radix`, icons `lucide`). After adding, read the generated file and fix issues
  (e.g. `empty.tsx`/`sonner.tsx` here needed a manual `import * as React`).
- **shadcn styling rules (enforced):** use **semantic tokens** (`bg-muted`,
  `text-muted-foreground`), never raw colors (`bg-blue-500`); `gap-*` not `space-y-*`;
  `size-*` when w=h; icons in components use `data-icon` (no size classes); use `cn()`
  for conditional classes; use components (`Badge`, `Empty`, `Separator`…) over custom
  markup. Category accent colors are the one intentional exception (dynamic, applied via
  inline `style`).
- **Tailwind v4:** theme tokens + custom CSS (the ambient `drift` animation, base
  resets) live in `src/style.css`. Don't add a `tailwind.config.js`.
- **Gmail scripts are read-only.** `scripts/gmail/credentials.json` and `token.json`
  are git-ignored — never commit them.

## Common changes

- **Add a UI component:** `pnpm dlx shadcn@latest add <name>`, then compose it in a
  feature component under `components/`.
- **Add a route/page:** create `src/pages/<x>.tsx`, wire it in `resolve()` in
  `src/App.tsx`, link with `<Link to="…">` from `lib/router`.
- **Add a category:** add an entry to `CATEGORIES` in `lib/edition.ts` (id + label +
  oklch accent), then map an icon in `components/site/category-badge.tsx`.
- **Change the data schema:** update types **and** validators in `lib/edition.ts`, the
  loader in `lib/content.ts`, the `scripts/newsletter/validate.ts` gate, **and**
  `INSTRUCTION.md` (the agent writes to this contract). Re-run `pnpm newsletter:validate`.
- **Rename/restyle the product:** `lib/site.ts` (name, tagline, description, author,
  locale, links) + `index.html` `<title>`/meta.
