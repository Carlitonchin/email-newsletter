# `content/` — the data the website is built from

Everything in this folder is **generated by the daily agent**. The website bundles
it at build time (`import.meta.glob`), so adding a new edition is just: drop a JSON
file here, run `pnpm newsletter:validate`, then `pnpm build`.

```
content/
├── editions/
│   └── YYYY-MM-DD.json   ← one file per day (≤ 5 summaries each)
└── state.json            ← bookkeeping: last processed email timestamp
```

- **`editions/<date>.json`** — the digest for one day. Filename must equal the
  `date` field. Schema and rules live in [`lib/edition.ts`](../lib/edition.ts) and
  are documented step-by-step in [`INSTRUCTION.md`](../INSTRUCTION.md).
- **`state.json`** — remembers `lastProcessedTimestamp` (epoch seconds) so the next
  run only looks at emails newer than the last one it summarized.

> `2026-06-26.json` is **sample data** shipped with the project to show the design.
> Delete it before (or on) your first real run.

Validate anytime with:

```bash
pnpm newsletter:validate
```
