# `content/` — the data the website is built from

Everything here is **generated by the daily agent**. The website bundles it at build
time (`import.meta.glob`), so publishing a new day is just: add files under a dated
folder, run `pnpm newsletter:validate`, then `pnpm build`.

```
content/
├── editions/
│   └── YYYY-MM-DD/          ← one folder per day
│       ├── index.json       ← day metadata + ordered article slugs (≤ 5)
│       └── <slug>.json      ← one article each (file name = slug)
└── state.json               ← bookkeeping: last processed email timestamp
```

- **`editions/<date>/<slug>.json`** — one article (a blog post + quiz). The file name
  must equal its `slug`, and its `date` must equal the folder name.
- **`editions/<date>/index.json`** — the day's metadata: `date`, `generatedAt`,
  optional `title`/`intro`, and `articles` (the slugs **in display order**). Every
  listed slug must have a file and vice versa.
- **`state.json`** — remembers `lastProcessedTimestamp` (epoch seconds) so the next
  run only looks at emails newer than the last one it processed.

Schema + rules live in [`lib/edition.ts`](../lib/edition.ts); the full agent
playbook is [`INSTRUCTION.md`](../INSTRUCTION.md).

> `editions/2026-06-26/` is **sample data** showcasing the design (note the
> first-person TypeScript article). Delete the folder before your first real run.

Validate anytime with:

```bash
pnpm newsletter:validate
```
